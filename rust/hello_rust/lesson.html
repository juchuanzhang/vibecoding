🧩 逐段讲解

1️⃣ 变量与可变性

let x = 5; let mut y = 10; y += 1;

关键点：

  • let 声明的变量默认不可变（immutable）
  • 加 mut 关键字才可变 —— 这是 Rust 的设计哲学：显式优于隐式
  • 打印字符串中 {x} 是直接插值语法
⚡ Rust 特色： 默认不可变，减少意外修改。相比其他语言默认可变，这是一个重要区别。

2️⃣ 基本数据类型

类型说明示例
i32有符号 32 位整数-42
u32无符号 32 位整数100
f6464 位浮点数（双精度）3.14
bool布尔值true
char字符，4 字节，支持 Unicode'R', '🦀'
&str字符串切片（不可变引用）"hello"
💡 Rust 的 char 是 4 字节（UTF-32），不像 C 语言是 1 字节。可以存任何 Unicode 字符。

3️⃣ 字符串（String）

let mut s = String::from("Hello"); s.push_str(", Rust!");

Rust 有两种字符串类型：

  • &str —— 不可变字符串切片（编译期已知，类似字符串字面量）
  • String —— 可变、堆分配的字符串（运行时动态增长）
⚡ 所有权： String 拥有数据，超出作用域自动释放内存。没有 GC，没有手动 free。

4️⃣ 函数

fn add(a: i32, b: i32) -> i32 { a + b }
  • 必须声明参数类型和返回类型
  • 最后一个表达式是返回值（没有分号）
  • 如果想提前返回，用 return 关键字

5️⃣ if/else

if grade >= 60 { ... } else { ... }

与大多数语言类似，但注意：条件不需要括号，但代码块必须用花括号。

6️⃣ for 循环

for i in 0..3 { ... }

0..3 是一个 Range（范围），左闭右开，产生 0, 1, 2。

7️⃣ match 匹配

match number { 1 => ..., 2 => ..., _ => ..., // 通配符，匹配所有其余情况 }

match 是 Rust 的王牌 —— 必须穷举所有可能性！_ 是通配符。

8️⃣ 向量（Vec）

let mut v = vec![1, 2, 3]; v.push(4);

Vec<T> 是动态数组，vec! 是创建向量的宏。

9️⃣ Option 与错误处理

fn divide(a: f64, b: f64) -> Option<f64> { if b == 0.0 { None } else { Some(a / b) } }
  • Option<T> 表示「可能有值，可能没有」—— 没有 null！
  • Some(value) = 有值，None = 无值
  • 用 match 解包，编译器强制你处理 None 的情况
⚡ 没有 Null 指针异常！ Rust 用 Option<T> 替代 null 指针，必须在编译期处理 None 的情况，彻底告别空指针异常。

🎯 Rust 核心概念总结

概念说明
所有权 (Ownership)每个值有唯一所有者，离开作用域自动释放。无需 GC，无需手动管理内存。
借用 (Borrowing)&T 不可变借用，&mut T 可变借用。编译时检查数据竞争。
表达式 (Expression)if、match、函数体都是表达式，可以返回值。
模式匹配 (Pattern Matching)match 必须穷举所有分支，与 Option/Result 结合非常强大。
零成本抽象 (Zero-Cost Abstraction)高级特性的运行时开销为零，无 GC、无虚函数表开销。

cargo run编译并运行

cargo test运行测试
cargo doc --open生成本地文档并打开
