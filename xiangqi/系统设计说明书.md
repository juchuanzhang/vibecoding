# 象棋分析软件（XiangQi Analyzer）系统设计说明书

## 1. 引言

### 1.1 系统概述

XiangQi Analyzer v0.2 采用 Python 后端 + WebSocket 前端架构。Python 引擎支持多线程并行搜索，通过 WebSocket 实时推送分析进度，前端永不冻结。

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| WebSocket | 全双工实时通信协议，服务器可主动推送消息 |
| aiohttp | Python 异步 HTTP/WebSocket 服务器框架 |
| ThreadPoolExecutor | Python 多线程执行器，并行搜索根走法 |
| Null Move Pruning | 跳过走棋方一步来快速获得评估下界 |
| Late Move Reduction | 后几个走法减少搜索深度 |
| Quiescence Search | 静态搜索，继续搜索吃子走法避免水平线效应 |
| Iterative Deepening | 迭代加深，从浅层逐步加深搜索 |
| Lazy SMP | 多线程各自独立迭代加深搜索 |

### 2.2 通信机制

前端与后端通过 WebSocket 实时通信：

```
┌──────────┐   WS连接   ┌──────────┐   多线程搜索   ┌──────────┐
│ browser  │ ←────────→ │server.py │ ──────────→ │ 线程池    │
│ (Canvas) │            │(aiohttp) │              │(N cores) │
│          │ ←推送结果→ │          │ ←────────── │          │
└──────────┘            └──────────┘              └──────────┘
```

**消息格式（JSON）**：

请求（前端 → 后端）：
```json
{"type": "move", "from": [x,y], "to": [x,y]}
{"type": "analyze", "depth": 8}
{"type": "undo"}
{"type": "new_game"}
{"type": "import_fen", "fen": "..."}
```

响应（后端 → 前端）：
```json
{"type": "board_state", "pieces": [...], "side": "red", ...}
{"type": "analysis_progress", "depth": 4, "best_move": ..., "score": 120}
{"type": "analysis_complete", "depth": 8, "best_move": ..., "candidates": [...], ...}
{"type": "error", "message": "..."}
```

### 2.3 多线程搜索设计

**Lazy SMP 方案**：N 个线程各自独立进行迭代加深搜索，共享全局置换表。

```
analyze(depth=8) 流程：
1. 主线程搜索 depth 1-2（快速，< 0.1秒）
2. 推送浅层结果到前端（progress消息）
3. 启动 N 个工作线程，各自从 depth 3 迭代加深到 depth 8
4. 每个线程完成一个深度后，更新全局置换表
5. 主线程定期检查各线程结果，推送最优到前端
6. 所有线程完成后，推送最终结果
```

### 3.7 server.py — aiohttp 服务器模块

```python
from aiohttp import web

async def websocket_handler(request):
    ws = web.WebSocketResponse()
    await ws.prepare(request)
    game = GameSession()
    async for msg in ws:
        data = json.loads(msg.data)
        if data['type'] == 'move':
            result = game.make_move(data['from'], data['to'])
            await ws.send_json(result)
        elif data['type'] == 'analyze':
            # 异步启动分析，渐进推送
            await game.analyze_and_push(ws, data['depth'])
    return ws

app = web.Application()
app.router.add_static('/', './')  # 静态文件
app.router.add_get('/ws', websocket_handler)
```

## 5. 性能优化设计

### 5.1 多线程优化

| 优化项 | 方法 | 预期效果 |
|--------|------|---------|
| Lazy SMP | N线程各自迭代加深 | 4核≈3-4倍加速 |
| 全局置换表 | 线程间共享TT | 避免重复搜索 |
| Null Move Pruning | 跳过走棋方一步 | 减少30-50%节点 |
| Late Move Reduction | 后几个走法减深度 | 减少10-20%节点 |
| Quiescence Search | 静态搜索 | 防止水平线效应 |

### 5.2 前端优化

| 优化项 | 方法 | 预期效果 |
|--------|------|---------|
| WebSocket 渐进推送 | 浅层结果先展示 | 用户立即看到推荐 |
| Canvas 渲染 | 仅重绘变化区域 | 60fps流畅 |
| 异步通信 | WS双向实时 | 分析时UI完全可用 |
