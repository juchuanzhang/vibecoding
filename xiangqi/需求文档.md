# 象棋分析软件（XiangQi Analyzer）需求文档

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| 名称 | XiangQi Analyzer（象棋分析器） |
| 类型 | Web 应用（Python 后端 + JS 前端） |
| 部署 | Python HTTP+WebSocket 服务器，浏览器前端 |
| 目标 | 提供象棋棋盘界面 + 局势评分分析 + 最优走法推荐，充分利用多核 CPU |

## 2. 技术架构

### 2.1 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| AI引擎 + 规则引擎 | Python | 高性能搜索引擎，支持多线程并行搜索 |
| 后端服务器 | Python (aiohttp) | HTTP 静态文件 + WebSocket 实时通信 |
| 前端通信层 | JS WebSocket | 实时接收分析结果，渐进式展示 |
| 棋盘UI | HTML / CSS / JS (Canvas) | 渲染棋盘、棋子、交互 |

### 2.2 架构变更说明

v0.1 采用 Rust+WASM 纯前端方案（已打 tag `v0.1-wasm`），存在以下问题：
- WASM 运行在浏览器单线程，无法利用多核 CPU
- 深层搜索（6层以上）导致 UI 长时间冻结
- 置换表等搜索状态无法跨 Worker 共享

v0.2 改为 Python 后端方案，优势：
- Python 原生支持多线程（ThreadPoolExecutor），可利用全部 CPU 核心
- WebSocket 实时推送分析进度，前端永不冻结
- 后端可维护全局置换表，多请求间共享搜索缓存
- 搜索深度不再受限，8-10层常规可达

## 3. 功能需求

### 3.1 棋盘界面

- 标准 9×10 中国象棋棋盘（Canvas 渲染）
- 32 枚棋子，初始标准开局布局，中文字符显示
- 红方棋子红色，黑方棋子黑色
- 走棋方式：鼠标点击选择棋子 → 点击目标位置走棋
- 选中棋子高亮，上一步走法标记，合法目标位置提示

### 3.2 走棋规则引擎（Python 实现）

完整实现所有棋子走法规则：

| 棋子 | 规则要点 |
|------|---------|
| 帅/将 | 九宫内移动，每次一步（上下左右），不能对面（飞将） |
| 仕/士 | 九宫内斜走一步 |
| 相/象 | 走田字，不能过河，塞象眼不可走 |
| 马 | 赌马步，蹩马腿不可走 |
| 车 | 直线任意步数，不可越子 |
| 炮 | 直线任意步数，吃子需隔一子（翻山） |
| 兵/卒 | 未过河只能前进，过河后可左右，不能后退 |

其他规则：
- 走棋后不能使己方被将军（送将禁止）
- 自动检测将军状态、绝杀/困毙

### 3.3 局势评分（核心功能）

评分体系与 v0.1 相同：
- 总分 = 红方评分 - 黑方评分
- 子力基础分 + 位置加分 + 王安全加分 + 机动性加分
- 实时显示评分数值 + 可视化进度条 + "红方领先/黑方领先"文字

### 3.4 最优走法推荐（核心功能）

#### 搜索算法

- **Minimax + Alpha-Beta 剪枝 + 迭代加深**
- **Null Move Pruning**：跳过走棋方的一步来快速获得评估下界
- **Late Move Reduction**：后几个走法减少搜索深度
- **Quiescence Search**：静态搜索防止水平线效应
- **多线程并行搜索**：ThreadPoolExecutor，N个线程分别搜索不同根走法

#### 搜索深度

- 可调：1-10 层（默认 4 层）
- 渐进式分析：先 2 层快速结果，再深搜到目标深度

#### 性能预期（4核 CPU）

| 搜索深度 | 单线程耗时 | 4线程并行耗时 |
|---------|-----------|-------------|
| 4 层 | < 2 秒 | < 1 秒 |
| 6 层 | < 10 秒 | < 5 秒 |
| 8 层 | < 60 秒 | < 20 秒 |
| 10 层 | < 5 分钟 | < 2 分钟 |

#### 推荐展示

- 推荐走法在棋盘上用**不同颜色箭头**标记（绿色粗=最优，黄色中=次选，橙色细=第三）
- 分析面板显示：推荐走法中文描述、评分变化、推演路径、Top 3 候选
- WebSocket 实时推送：浅层结果先展示，深层结果更新

### 3.5 交互功能

| 功能 | 说明 |
|------|------|
| 新局 | 重置棋盘为标准开局 |
| 悔棋 | 撤销上一步走棋 |
| 分析 | 点击触发局势评分 + 最优走法推荐 |
| 搜索深度设置 | 下拉选择 1-10 层 |
| 走棋历史 | 列表显示所有走棋记录（中文描述） |
| 导入棋局 | FEN 格式导入 |
| 导出棋局 | 导出当前棋局 FEN |

### 3.6 通信机制

- **走棋操作**：WebSocket 发送走棋请求 → 后端执行 → 返回棋盘状态
- **分析请求**：WebSocket 发送分析请求 → 后端多线程搜索 → 渐进式推送结果
- **实时更新**：后端每完成一个深度就推送结果，前端立即渲染

## 4. UI 设计

与 v0.1 相同，左侧棋盘 + 右侧分析面板 + 底部控制栏。

## 5. 开发与部署

### 5.1 开发环境要求

- Python 3.10+（运行引擎和服务器）
- aiohttp（WebSocket 服务器）
- 现代浏览器（Chrome/Firefox/Edge）

### 5.2 启动方式

```bash
python server.py
# 或
start.bat  # 自动启动
```

### 5.3 部署

- 服务器默认监听 `http://localhost:8080`
- 可部署到任意有 Python 的服务器

## 7. 性能优化策略

- Alpha-Beta 剪枝 + Null Move Pruning + Late Move Reduction
- 置换表（全局共享，跨请求缓存）
- 走法排序：TT最佳走法优先 + MVV-LVA + 杀手走法 + 历史启发
- 多线程并行搜索根走法
- WebSocket 渐进式推送，浅层结果先展示
- 迭代加深，每层利用上一层的 TT 结果优化排序
