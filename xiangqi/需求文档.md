# 象棋分析软件（XiangQi Analyzer）需求文档

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| 名称 | XiangQi Analyzer（象棋分析器） |
| 类型 | Web 前端应用（Rust + WASM 高性能引擎） |
| 部署 | 纯前端，无需后端服务器，浏览器直接运行 |
| 目标 | 提供象棋棋盘界面 + 局势评分分析 + 最优走法推荐 |

## 2. 技术架构

### 2.1 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| AI引擎 + 规则引擎 | Rust → WASM | 高性能核心，接近原生速度 |
| 前端调用层 | JS（wasm-pack 生成的绑定） | 前端通过 JS 调用 WASM 接口 |
| 棋盘UI | HTML / CSS / JS | 渲染棋盘、棋子、交互 |
| 异步计算 | Web Worker | AI 计算在后台线程，不阻塞 UI |


## 3. 功能需求

### 3.1 棋盘界面

- 标准 9×10 中国象棋棋盘
- 含河界标注（楚河汉界）、九宫格斜线
- 32 枚棋子，初始标准开局布局
- 棋子使用中文字符显示：

| 红方 | 黑方 |
|------|------|
| 帅 | 将 |
| 仕 | 士 |
| 相 | 象 |
| 马 | 马 |
| 车 | 车 |
| 炮 | 炩 |
| 兵 | 卒 |

- 红方棋子红色，黑方棋子黑色
- 走棋方式：鼠标点击选择棋子 → 点击目标位置走棋
- 选中棋子高亮，上一步走法标记（起点/终点）
- 合法目标位置提示（选中棋子后显示可走位置）

### 3.2 走棋规则引擎（Rust 实现）

完整实现所有棋子走法规则：

| 棋子 | 规则要点 |
|------|---------|
| 帅/将 | 九宫内移动，每次一步（上下左右），不能对面（飞将） |
| 仕/士 | 九宫内斜走一步 |
| 相/象 | 走田字，不能过河，塞象眼不可走 |
| 马 | 赌马步，蹩马腿不可走 |
| 车 | 直线任意步数，不可越子 |
| 炮 | 直线任意步数，吃子需隔一子（翻山） |
| 兵/卒 | 未过河只能前进，过河后可左右，不能后退 |

其他规则：
- 走棋后不能使己方被将军（送将禁止）
- 自动检测将军状态，提示被将
- 自动检测绝杀/困毙，判定胜负

### 3.3 局势评分（核心功能）

#### 评分体系

总分 = 红方评分 - 黑方评分（正数 = 红方优势，负数 = 黑方优势）

**子力基础分**：

| 棋子 | 基础分 |
|------|--------|
| 帅/将 | 10000（特殊，不可被吃） |
| 车 | 900 |
| 炮 | 450 |
| 马 | 400 |
| 仕/士 | 200 |
| 相/象 | 200 |
| 兵（未过河） | 100 |
| 兵（已过河） | 200 |
| 卒（未过河） | 100 |
| 卒（已过河） | 200 |

**位置加分**：
- 每种棋子在棋盘不同位置有专属加分表（如马在中心加分，车在开放线加分）
- 兵/卒过河后位置加分显著增加

**机动性加分**：
- 棋子可走位置数量 × 权重系数

#### 评分展示

- 右侧分析面板实时显示当前评分（数值 + 可视化进度条）
- 评分范围：正数红色表示红优，负数黑色表示黑优，0 表示均势

### 3.4 最优走法推荐（核心功能）

#### 搜索算法

- **Minimax + Alpha-Beta 剪枝**（Rust 实现，WASM 运行）
- 搜索深度可调：1-6 层（默认 4 层）
- 记忆化优化：Transposition Table（哈希表缓存已评估局面）

#### 性能预期

| 搜索深度 | WASM 预期耗时 |
|---------|-------------|
| 3 层 | < 0.5 秒 |
| 4 层 | < 2 秒 |
| 5 层 | < 5 秒 |
| 6 层 | < 15 秒 |

#### 推荐展示

- 推荐走法在棋盘上用**箭头标记**（起点 → 终点）
- 分析面板显示：
  - 推荐走法描述（如"车八平五"）
  - 评分变化（走此步后评分预估）
  - 前几步推演过程（搜索树关键路径）
- 多个候选走法按评分排序展示（Top 3）

### 3.5 交互功能

| 功能 | 说明 |
|------|------|
| 新局 | 重置棋盘为标准开局 |
| 悔棋 | 撤销上一步走棋 |
| 分析 | 点击按钮触发局势评分 + 最优走法推荐 |
| 搜索深度设置 | 下拉选择 1-6 层 |
| 走棋历史 | 列表显示所有走棋记录 |
| 导入棋局 | 支持简易 FEN 格式导入 |
| 导出棋局 | 导出当前棋局 FEN |

### 3.6 对弈模式

- **PVP（人对人）**：双人轮流走棋，红先
- **PVA（人对AI）**：人与 AI 对弈（后续版本可考虑）

## 5. 开发与构建

### 5.1 开发环境要求（仅开发时需要）

- Rust + rustup（编译引擎）
- wasm-pack（构建 WASM 产物）
- 任意浏览器（运行测试）

### 5.2 构建流程

```bash
# 构建 WASM 引擎
cd xiangqi-core
wasm-pack build --target web --out-dir pkg

# 前端直接打开 index.html 即可运行
```

### 5.3 部署

- `pkg/` 目录包含构建产物（WASM + JS 绑定）
- 用户/部署方无需安装 Rust，浏览器直接加载运行
- 可部署至任意静态服务器或 GitHub Pages

## 7. 性能优化策略

- Alpha-Beta 剪枝大幅减少搜索节点
- Transposition Table 缓存已评估局面，避免重复计算
- 走法排序：优先搜索吃子走法，提高剪枝效率
- Web Worker 后台计算，UI 不冻结
- WASM 接近原生执行速度（比 JS 快 5-10 倍）
